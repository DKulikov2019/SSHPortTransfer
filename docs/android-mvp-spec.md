# Техническое задание: Android MVP приложения SSHPortTransfer

## 1. Цель
Разработать Android-приложение на Kotlin для установки SSH-соединения с удалённым сервером и поднятия локального TCP-порта с переадресацией на удалённый адрес через SSH local port forwarding.

Приложение должно обеспечивать поведение, эквивалентное команде:

```bash
ssh -N -L 23070:127.0.0.1:23070 ginger@150.251.152.3 -p 47708
```

При этом приложение не должно содержать предзаполненных значений в UI-формах. Все параметры подключения пользователь вводит вручную или загружает из ранее сохранённого профиля.

## 2. Репозиторий
- Репозиторий: `DKulikov2019/SSHPortTransfer`
- Платформа в рамках данного этапа: только Android
- Язык разработки: Kotlin
- UI: Jetpack Compose

## 3. Область реализации MVP
В MVP необходимо реализовать:
- создание и редактирование профилей SSH-подключения;
- хранение одного или нескольких профилей подключения;
- ввод параметров SSH-соединения без предзаполнения полей;
- поддержку SSH-аутентификации по паролю;
- архитектурную готовность к добавлению аутентификации по приватному ключу;
- установку SSH-соединения;
- запуск local port forwarding;
- остановку SSH-соединения и проброса порта;
- отображение текущего статуса соединения;
- отображение ошибок подключения;
- удержание активного соединения через foreground service;
- сохранение секретов в защищённом хранилище;
- базовую валидацию host key fingerprint при первом подключении.

В MVP можно не реализовывать:
- iOS-версию;
- импорт/экспорт ключей;
- биометрию;
- несколько одновременных активных туннелей;
- сложную аналитику логов;
- внешний доступ к локальному порту за пределами устройства.

## 4. Основной пользовательский сценарий
1. Пользователь открывает приложение.
2. Создаёт новый профиль подключения.
3. Вручную вводит:
   - имя профиля;
   - SSH host;
   - SSH port;
   - username;
   - способ аутентификации;
   - пароль;
   - локальный адрес привязки;
   - локальный порт;
   - удалённый host;
   - удалённый порт;
   - keepalive interval;
   - флаг auto reconnect.
4. Сохраняет профиль.
5. Запускает подключение.
6. Приложение поднимает foreground service.
7. Приложение устанавливает SSH-соединение.
8. Приложение запускает local port forwarding.
9. Пользователь видит статус `Connected`.
10. При разрыве соединения приложение показывает ошибку и, если включено, выполняет переподключение.
11. Пользователь может вручную отключить туннель.

## 5. Функциональные требования

### 5.1. Управление профилями
Приложение должно позволять:
- создавать профиль подключения;
- редактировать существующий профиль;
- удалять профиль;
- просматривать список профилей;
- выбирать профиль для подключения.

### 5.2. Параметры профиля
Профиль должен содержать следующие поля:
- `id`;
- `name`;
- `sshHost`;
- `sshPort`;
- `username`;
- `authType`;
- `localHost`;
- `localPort`;
- `remoteHost`;
- `remotePort`;
- `keepAliveSeconds`;
- `autoReconnect`.

Требование к UI:
- при создании нового профиля поля формы должны быть пустыми или иметь только системные безопасные значения по умолчанию для переключателей/селекторов, но не должны содержать заранее подставленных серверов, логинов, адресов или портов.
- не использовать предзаполнение значений подключения, связанных с конкретным сервером.

### 5.3. Аутентификация
В MVP должна быть реализована аутентификация:
- по паролю.

Нужно предусмотреть расширение архитектуры для поддержки:
- приватного ключа;
- passphrase.

### 5.4. SSH-соединение
Приложение должно:
- устанавливать SSH-соединение по введённым параметрам;
- поддерживать соединение без открытия интерактивной shell-сессии;
- использовать SSH local port forwarding;
- поддерживать keepalive;
- корректно завершать соединение.

### 5.5. Port forwarding
Необходимо реализовать локальный проброс порта по модели:
- локальный `localHost:localPort` на Android-устройстве;
- удалённый целевой адрес `remoteHost:remotePort`, доступный с точки зрения SSH-сервера.

Приложение должно обеспечивать поведение, эквивалентное флагу `-L` в OpenSSH.

### 5.6. Статусы соединения
Приложение должно отображать состояния:
- `Disconnected`;
- `Connecting`;
- `Connected`;
- `Reconnecting`;
- `Failed`.

### 5.7. Обработка ошибок
Приложение должно корректно отображать ошибки:
- неверный host;
- неверный порт;
- ошибка аутентификации;
- недоступность сервера;
- отказ в пробросе порта;
- конфликт локального порта;
- ошибка host key verification;
- таймаут соединения.

### 5.8. Поведение в фоне
При активном туннеле приложение должно:
- запускать foreground service;
- отображать persistent notification;
- позволять остановить туннель из notification или из приложения;
- не зависеть от нахождения UI на экране для удержания соединения.

### 5.9. Безопасность
Приложение должно:
- не хранить пароль в открытом виде в Room или других обычных хранилищах;
- использовать защищённое хранилище Android для секретов;
- не выводить секреты в логи;
- при первом подключении показывать fingerprint удалённого host key;
- позволять пользователю подтвердить доверие к fingerprint;
- сохранять trusted host keys для последующих подключений.

## 6. Нефункциональные требования
- Язык: Kotlin.
- Минимальная архитектура: MVVM или MVVM с разделением на data/domain/presentation.
- UI должен быть реализован на Jetpack Compose.
- Приложение должно быть рассчитано на дальнейшее расширение.
- Код должен быть разделён на слои и не содержать SSH-логики внутри composable-функций.
- Компонент туннелирования должен быть отделён от UI.
- Поведение должно быть устойчивым к пересозданию Activity.

### 8.2. Domain model

#### ConnectionProfile
```kotlin
data class ConnectionProfile(
    val id: String,
    val name: String,
    val sshHost: String,
    val sshPort: Int,
    val username: String,
    val authType: AuthType,
    val localHost: String,
    val localPort: Int,
    val remoteHost: String,
    val remotePort: Int,
    val keepAliveSeconds: Int,
    val autoReconnect: Boolean
)
```

#### AuthType
```kotlin
enum class AuthType {
    PASSWORD,
    PRIVATE_KEY
}
```

#### TunnelState
```kotlin
sealed interface TunnelState {
    data object Disconnected : TunnelState
    data object Connecting : TunnelState
    data class Connected(
        val localHost: String,
        val localPort: Int,
        val remoteHost: String,
        val remotePort: Int
    ) : TunnelState
    data class Reconnecting(val attempt: Int) : TunnelState
    data class Failed(val message: String) : TunnelState
}
```

### 10.2. Экран редактирования профиля
Должен содержать вводимые поля:
- Profile name;
- SSH host;
- SSH port;
- Username;
- Auth type;
- Password;
- Local host;
- Local port;
- Remote host;
- Remote port;
- Keep alive;
- Auto reconnect.

Требование:

## 11. Хранение данных

### 11.1. Room
В Room хранить:
- метаданные профиля;
- настройки reconnect/keepalive;
- trusted host fingerprints или ссылку на них.

### 11.2. Secure storage
В защищённом хранилище хранить:
- пароль;
- в дальнейшем приватный ключ и passphrase.

## 12. Host key verification
При первом подключении к неизвестному серверу приложение должно:
- получить fingerprint host key;
- показать его пользователю;
- запросить подтверждение доверия;
- при согласии сохранить fingerprint;
- при несовпадении ранее сохранённого fingerprint блокировать подключение и отображать предупреждение.

## 13. Notification
При активном туннеле notification должна содержать:
- признак активного SSH-соединения;
- краткое описание проброса;
- действие Disconnect.

## 14. Требования к качеству кода
- Чёткое разделение ответственности по слоям.
- Отсутствие прямой SSH-логики в UI.
- Минимизация platform-specific логики вне service/data слоя.

## 15. Критерии приёмки MVP
MVP считается выполненным, если:
1. Пользователь может создать профиль без предзаполненных значений подключения.
2. Пользователь может сохранить профиль.
3. Пользователь может ввести параметры SSH-подключения вручную.
4. Приложение устанавливает SSH-соединение по указанным параметрам.
5. Приложение поднимает local port forwarding.
6. Активное соединение удерживается foreground service.
7. Пользователь видит актуальный статус соединения.
8. Пользователь может остановить соединение вручную.
9. Пароль хранится только в защищённом хранилище.
